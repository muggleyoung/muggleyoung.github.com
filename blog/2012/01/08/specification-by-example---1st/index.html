Specification by Example - 1st

What is specification by examples?

即使在一个team中，不同的人，语言组织和思维方式是不同的，这样在交流中总会多多少少存在误解。

比如，一个书店，买了10本以上书的顾客就是vip，那么恰好买了10本书的，是不是vip呢？在这一点上，可能就会出现不同的理解。这时候，我们使用例子来说明问题：

买书的数目    会员资格

9本          普通会员

10本         VIP

11本         VIP

这样，就在一定程度上降低了产生歧义的可能。

个人觉得，此处的理念和DDD(Data Driven Design), 另一个DDD(Domain Driven Design)有交汇的地方。

比如，在设计case的时候，同一个case，换成不同的数据，就可以测到很多个验证点；再比如，在团队中使用这样的方法，有助于分析师和开发在同一个page交流。

 

第一天，我们通过两个workshop进行了简单的练习，如何进行specification by examples。

第一个，体会specification是怎样在项目中帮助我们交流需求。有一张图非常简单的说明了问题。

转述就是：

examples elaborated requirements

examples can become test cases

And test cases is verify requirements
