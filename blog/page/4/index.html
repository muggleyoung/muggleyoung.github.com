Specification by Example - 1st

What is specification by examples?

即使在一个team中，不同的人，语言组织和思维方式是不同的，这样在交流中总会多多少少存在误解。

比如，一个书店，买了10本以上书的顾客就是vip，那么恰好买了10本书的，是不是vip呢？在这一点上，可能就会出现不同的理解。这时候，我们使用例子来说明问题：

买书的数目    会员资格

9本          普通会员

10本         VIP

11本         VIP

这样，就在一定程度上降低了产生歧义的可能。

个人觉得，此处的理念和DDD(Data Driven Design), 另一个DDD(Domain Driven Design)有交汇的地方。

比如，在设计case的时候，同一个case，换成不同的数据，就可以测到很多个验证点；再比如，在团队中使用这样的方法，有助于分析师和开发在同一个page交流。

 

第一天，我们通过两个workshop进行了简单的练习，如何进行specification by examples。

第一个，体会specification是怎样在项目中帮助我们交流需求。有一张图非常简单的说明了问题。

转述就是：

examples elaborated requirements

examples can become test cases

And test cases is verify requirements

 

第二个workshop难度增加后，引出了missing concept这个点。

当我们在写specification或者test case的时候，觉得逻辑越来越复杂的时候，就需要静下来好好想一想，是不是能找出什么missing concept，让我们把case简单化，分成两个不同的case分别验证。

2011年总结篇

这一年，我终于毕业了，真的是熬过了两年半的研究生，成功在3月答辩，顺利毕业。

紧接着，在毕业和入职的间隙，花了两周时间，厦门，上海，杭州，转了一大圈。并成功在和亲爱的飞碰头后一天，丢了相机，和这次毕业旅行的所有物理记忆。

四月入职，就来到了北京。跟着同事们在大大小小的项目中学习着。顺便学了下打太极这档子事儿。

在北京，也算是和某人关系稳定些了。稳定的生气，吵架。

8月，沾TWU的光，去了躺印度。在学习中，认识了负责的trainer，热情的同学们。感觉自己也建立起自信了，不那么害羞了。除了学习，每个周末大家还会集体出游，在泰姬陵低调的过了25岁生日。想念我的老师们，同学们，偶尔有那么一下下也想念印度的食物~

回到北京，崔同学就要结婚了。平生头一次当伴娘，看着崔被爸爸领着走的时候，我在拼命的憋眼泪。还一度被大伙威胁，再哭就不许做伴娘了。

2012的开始，回顾这一年，经历的事情不少，但缺少了以往坚持做的事情，比如规律的读书，更新blog，比如语言的学习。积极的一面是同样放弃浪费大量时间追各种美剧，电影，show还是要看的，但要适度的啦。

Jan 3rd, 2012

回顾–First Project

 

 

突然意识到不该再偷懒，当初培养坚持写博的好习惯多么不容易啊，多多少少继续写吧~

工作之后，好多东西顺手就记在了evernote上，乱糟糟一大堆。好处就是，现在随别找点东西改吧改吧就能贴出来了。

下面是在TWU的时候回想自己4个月的工作，回想自己做过的第一个项目的一些记录。
—————————————————————————————–

当我进第一个项目的时候，组里没有QA，而我做为刚入职的新人，对要做的工作近乎于一无所知。项目开发人员耐心解释部署是怎样的过程，教我使用基本的git命令，甚至还要关心我对于一个stroy有没有全面的test case。

但是被开发引导的测试，加上新手tester完全没有测试的sense，这时候我亲爱的buddy出场了！我们不在一个项目组，平时工作遇到问题，首先不方便交流，我咨询组里的同事更快捷，其次，buddy没有我们项目的context，太细节的问题也不好给出答案。于是她安排了一系列的小灶课程，每天午饭后，用半小时的时间跟我介绍QA一般都要做什么，怎么做；敏捷的QA在team中的作用等等，还分享很多资源给我学习。

（注：现在，我们已经有真正的对新入职QA同学的培训了，主要就讲敏捷基础什么的。）

当时的项目并不是很典型的敏捷项目，没有TDD，没有持续集成，没有mingle墙。钱钱在项目总结中说到的好与不好，对我而言只是概念而已，却很难真正理解为什么好，为什么不好，怎样才是好，怎样才是不好。

（昨天还跟同事聊起过，什么理论听起来都特别容易明白，但真正理解到可以清楚明白其中的道理，自己不经历折磨，摸索的过程，是很难真正理解的
现在，虽然我没怎么写过，也不怎么会写TDD，却真正明白了他的重要性，这个聊起后面的项目时候可以说说；还有mingle和物理墙的管理，看似没什么技术含量，但现在也真正明白对于项目，为什么要维护他们，为什么维护总是会被人忘记。）

当时buddy的课程也是被我当作全新的概念在学习，比较少有实践和体会。一个多月的时间，最大的收获是技术的学习，从QA到开发到最后的support阶段，了解了项目中的不同角色都做些什么事情。

这期间，除了快速的上手工作以外，体验support也是难得的经历。紧急修复小bug，处理用户使用中出现的问题，尤其是当系统有问题，客户发飙的时候怎么处理，对自己都是很好的锻炼。第一次被骂时，快哭了的场景，现在想想，也是珍贵的经验了。

最后，感谢又当爹又当妈事无巨细的同事们。耐心的讲解，项目进度不紧张时带我pair，给我机会了解了很多在大项目中，qa接触不到的事情。

给亲爱的王不拔

亲爱的王不拔小姐：

    签证被拒那天我心情也很差，所以才会想寄软香酥给你。这确实是一个没那么好的机会，但我一直认为you deserve it，好比你那200美刀的奖金。即使是对你3年工作的安慰奖，那也总能让人心理稍稍舒坦些。

    对我来说，研究生3年实在过的不满意。那天跟小冯同学说起这事儿，我说到了这3年过得多么没有意义，不快乐，小冯说这就是你最大的收获，知道自己根本不想要这样的生活。我一下释然了。

    作为新鲜人，或许我表达了太多对于工作的兴奋。常有同事问，工作了感觉怎么样。我都干脆的回答，比上学时舒坦多了。可不是么，找到一个无比欢乐的工作环境，喜欢的事情，觉得生活变得有意义，充实起来，我打心眼里的开心舒坦，恨不得见人就说我有多喜欢这份工作。想想你充实工作的3年，我是怎么熬过来的，心里会平衡些吗。

    其实你不必被我的状态影响，每个人处于每个阶段都有不同的快乐点。现在每天去健身房不也是很快乐的事情么。

    关于北京，我在这里出差，什么都不用操心，自然过得滋润。即使薪水两倍于我的同学，在北京生活的质量和尊严感也没法和我比，而积攒的银子并没有比我多出多少。你该明白我的意思吧。至少我现在不担心房租，住在二环边上，上班走走就到，偶尔向你们抱怨下吃的不如家里好吃，但也没亏着自己。可是你一个人来北京生活，那就不一样了。房租很贵，即使这样还有可能要坐半小时以上的地铁去上班。我目前能看到的结果，要么经济不大宽裕，要么生活质量无法满足我们这类娃的要求。

     鉴于我对杭州极其极其不稀罕，所以北京可以算是一个just so so的选择。那天的聊天，只是想告诉你生活两个月后我对这个地方产生的一些顾虑。你来了，我们俩会有更多机会海吃闲逛八卦呢。所以放心吧，你的决定我自然会尽力帮忙。

     我们是生活在现实中的理想主义者吧。我从来没有为你的大龄剩女问题担心过。我们没有过多的要求，不会为喜欢的人加上经济条件的限制，只是想找到一个让我们感到自在开心的娃而已，对吧。

实在想不起你还写了什么给我，白天攒了一肚子的话，到这时也变得没有逻辑没有条理。总之吧，我喜欢现在有追求充实的生活状态，我希望你尽快从现在的坏状态中解脱，所以有什么需要帮忙的，尽管开口呗。你的闺蜜只是刀子嘴而已。

                                                                                 你亲爱的杨也尼，葛郎台
