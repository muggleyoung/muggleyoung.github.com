什么是CSS reset？稀里糊涂倒腾了一阵css之后的我表示，压根儿没听过这是个啥....

其实呢，就是css的style中，有很多都有默认的一些数值设置，而这些设置在不同的浏览器中会显示不同的样式，这样就造成了浏览器之间的显示不一致问题。使用css reset 就是为了解决这个问题，让网页在不同的浏览器间有一致的表现。

比如，我遇到的问题很简单，设置了body的背景颜色，可是发现上下左右似乎总有那么5px的白边，h1上下总是有几个像素margin，搞不懂是为啥，自己重新设置了margin和padding，就好了。

看了一些介绍，似乎很不鼓励直接用

* { padding: 0; margin: 0; border: 0}

这种暴力的方法设置。说是浪费资源，大概会有性能问题。

常见的两种设置方法是YUI和Eric。

YUI，是Yahoo整出来的，据说深受喜爱。

body,div,dl,dt,dd,ul,ol,li,h1,h2,h3,h4,h5,h6,pre, 
form,fieldset,input,textarea,p,blockquote,th,td { 
padding: 0; 
margin: 0; 
} 
table { 
border-collapse: collapse; 
border-spacing: 0; 
} 
fieldset,img { 
border: 0; 
} 
address,caption,cite,code,dfn,em,strong,th,var { 
font-weight: normal; 
font-style: normal; 
} 
ol,ul { 
list-style: none; 
} 
caption,th { 
text-align: left; 
} 
h1,h2,h3,h4,h5,h6 { 
font-weight: normal; 
font-size: 100%; 
} 
q:before,q:after { 
content:''; 
} 
abbr,acronym { border: 0; 
}

我能看到的区别似乎是后者一次性重新定义了更多的元素，不知道在真正的使用中会有怎样的差别。

也许根据需要的不同，也可自定义reset吧。
